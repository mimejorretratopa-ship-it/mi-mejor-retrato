OBJETIVO PRINCIPAL:
Crear una aplicación extremadamente fácil de:
- desarrollar
- probar localmente
- debuggear
- mantener
- deployar en Netlify

REQUERIMIENTO CRÍTICO:
La app DEBE estar diseñada desde el inicio para soportar persistencia futura REAL sin necesidad de reescribir el frontend.

IMPORTANTE:
Aunque inicialmente la persistencia sea:
- localStorage
- mock JSON
- datos fake

la arquitectura DEBE permitir luego conectar fácilmente:
- Netlify Functions
- Supabase
- Firebase
- APIs REST
- backend real

SIN reescribir la UI.

REQUISITOS ARQUITECTURALES OBLIGATORIOS:

1. Separación estricta entre:
   - UI
   - estado
   - persistencia
   - lógica de negocio

2. Toda persistencia debe pasar por:
   - storage.js
   o
   - api.js

3. La UI NUNCA debe:
   - usar localStorage directamente
   - usar fetch directamente
   - modificar estado persistente directamente

4. Toda operación de datos debe ser async desde el inicio.

Ejemplo obligatorio:

```js
await storage.saveItem(data)
const items = await storage.loadItems()
````

NO hacer:

```js
localStorage.setItem(...)
fetch(...)
```

directamente en UI.

5. Diseñar desde el inicio usando adapter pattern simple.

Ejemplo conceptual:

```js
const storage =
    ENV === 'local'
      ? localStorageAdapter
      : apiAdapter;
```

6. Mantener arquitectura SIMPLE.
   NO overengineering.

7. Priorizar:

* claridad
* simplicidad
* mantenibilidad
* debugging fácil

sobre:

* patterns complejos
* abstracciones enterprise
* optimización prematura

8. Compatibilidad total con:

* Live Server
* VS Code
* Netlify static hosting

9. Todo debe funcionar SIN build step.

10. Usar estructura de archivos clara y mínima.
